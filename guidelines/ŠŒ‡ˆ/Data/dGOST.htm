Лабораторная работа

Отечественный стандарт цифровой подписи ГОСТ Р 34.10-94

 

Цель работы: изучение стандарта цифровой подписи ГОСТ Р 34.10-94 и освоение практических методов реализации на языке программирования схем постановки и проверки цифровых подписей на данном стандарте.

 

Описание работы.

Данный алгоритм цифровой подписи концептуально близок к алгоритму DSA. Стандарт вступил в действие с начала 1995 г. В настоящее время этот стандарт заменен на схему получения цифровой подписи с помощью эллиптических кривых.

Криптостойкость данного стандарта базируется на трудности вычисления дискретных логарифмов.

Опишем алгоритм.

Для подписания некоторого сообщения M пользователь A выполняет следующие шаги.

1. Получение параметров алгоритма. Выбираем:

- простое большое число р размерностью [509; 512] или [1020;1024] бит;

- простой сомножитель q  для числа (р - 1) размерностью [254; 256] бит (по крайней мере больше двух);

- а - число, удовлетворяющее условиям:

;

 

 - х – секретный ключ, меньший q;

- открытый ключ у = ах mod p.

2. Нахождение значения хэш от сообщения: m = H(M) mod q. Если m равно нулю, то устанавливают m = 1.

Для того чтобы подписать сообщение М, хешируется M с помощью хэш-функции h(M) в целое число m, m = h(M), 0<m<(p-1). В качестве h(M) применяется режим имитовставки алгоритма ГОСТ 28147-89. В рамках данной лабораторной работы, можно использовать операцию возведения в степень и операцию xor (исключающее или): возведение блока открытого текста в степень, являющуюся открытым ключом y, и сложение по xor с предыдущим значением хэш: m = (m Å (MiY) mod p) mod p, при начальном m = 0.

3. Генерируется случайное число k, причем 0 < k < q.

4. Вычисление составляющих электронной цифровой подписи (ЭЦП):

r = (аk mod p) mod q,

s = (x*r + k*m)mod q.

Если r оказалось равным нулю, то повторяются пункты 3 (заново генерируется число k) и 4.

5. Распространение открытых параметров p, q, y и а, r и s. Числа х и k являются сек­ретными.

На стороне B, осуществляющей проверку подписи, выполняются следующие вычисления.

v = mq-2 mod q,

z1 = (s * v) mod q,

z2 = ((q - r) * v) mod q,

u = ((aZ1 *yZ2) mod p) mod q.

Если u = r, то подпись считается верной.

 

Пример. Поставить цифровую подпись на текст «ЭБФ» с помощью стандарта ГОСТ Р 34.10-94.

Действия стороны A, ставящей подпись.

1. Выбираем модуль p = 31, q = 5.

Для q выполняются условия: q > 2 и ((p-1) mod q) = 0.

2. Задаем a = 16, удовлетворяющее условиям

.

 

3. Вычисляем по секретному ключу x = 3 (x < q) открытый ключ y = ax mod p = 163 mod 31 = 4.

4. Задаем число k = 4 (k < q).

5. Вычисляем r = (ak mod p) mod q = 164 mod 31 mod 5 = 2. Если r = 0, то необходимо задать новое k.

6. Вычисление хеш. Пусть m = 0. Тогда

- при M0 = 30: m = m ÅM0Y mod p = (0 Å 304 mod 31) mod 31 = (0 Å 1)mod 31 = 1.

- при M1 = 2: m = m ÅM1Y mod p = (1 Å 24 mod 31) mod 31 = (1 Å 16) mod 31= 17.

- при M2 = 21: m = m ÅM2Y mod p = (17 Å 214 mod 31) mod 31 = (17 Å 18) mod 31= 3.

Хэш равно m = 3. Если бы (m mod q) = 0, то приняли бы m = 1.

7. Вычисление s = (x*r + k*m)mod q = (3*2 + 4*3) mod 5 = 3.

8. Передача стороне B чисел p, q, a, y, r, s и открытого текста M.

Действия стороны B (проверка подписи).

1. Вычисление хэш из открытого текста, аналогично пункту 6 стороны А: m = 3.

2. Вычисление v = mq-2 mod q = 33 mod 5 = 2,

z1 = (s * v) mod q = (3*2) mod 5 = 1,

z2 = ((q - r) * v) mod q = ((5-2)*2) mod 5 = 1,

u = ((aZ1 *yZ2) mod p) mod q = ((161 * 41) mod 31) mod 5 = ((16*4) mod 31) mod 5 = 2.

3. Проверка: если u = r = 2, то подпись верна, иначе, неверна.

 

Дополнительные примеры: выполните постановку подписи для первых букв своих фамилии, имени и отчества.

 

Задание.

В данной лабораторной работе требуется освоить работу алгоритма цифровой подписи ГОСТ. Для этого необходимо выполнить следующее (каждый студент индивидуально выполняет задание и оформляет отчет).

2. Программно реализовать (для чисел размерностью в машинное слово) данный алгоритм в виде программы на любом из следующих языков программирования: C, С++, Pascal. Текст программы должен содержать комментарии к каждой строке кода.

3. Оформить отчет о выполненной работе, с приведением расчета из пункта 1, листингов программ и двух-трех тестов (как в первом пункте задания).

4. Выполнить сдачу работы руководителю лабораторного занятия (при этом существует возможность задания дополнительных вопросов).
