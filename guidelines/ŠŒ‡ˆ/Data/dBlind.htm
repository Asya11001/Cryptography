Лабораторная работа

Слепая цифровая подпись

 

Цель работы: освоение практических методов использования и собственной программной реализации алгоритма постановки слепой подписи.

 

Описание работы.

В алгоритмах слепой подписи (подпись вслепую) - сторона B подписывает (подтверждает подлинность) документ, присланный стороной A, без рассмотрения самого содержания документа.

Процесс постановки слепой подписи под документом можно рассматривать как помещение документа в конверт, а процесс удаления маскирующего элемента – как вскрытие конверта. Документ в запечатанном конверте прочитать нельзя. При постановке подписи на конверте, автоматически, с помощью копировальной бумаги, помещенной в конверт, на документе прописывается копия подписи.

Для исключения возможности мошенничества против подписывающей стороны, часть документов досматривается.

Отправитель А посылает порцию информации стороне В, которую В подписывает и возвращает А. Используя полученную подпись, сторона А может сама, с помощью вычислений, поставить подпись стороны В на более важном для себя сообщении m. По завершении этого протокола сторона В ничего не знает ни о сообщении m, ни о подписи под этим сообщением.

Слепые подписи обладают следующими свойствами:

1. В постановке подписи участвует две стороны.

2. Подпись другой стороны обладает всеми свойствами обычных цифровых подписей. Подпись, поставленная на маскирующем элементе документа, переносится на сам документ.

3. Подписывающее лицо В не может ознакомиться с содержимым документа стороны А, который он подписывает, и, в дальнейшем, не может связать подписанный документ с процессом подписания документа.

При организации анонимных безналичных расчетов клиент банка A не хочет раскрывать банку B информацию о том, какую сумму денег он хочет потратить. Когда сообщение m с подписью sB(m) предъявляется банку В для оплаты, банк В не может проследить, кто именно из клиентов предъявляет подписанный документ. Это позволяет пользователю А остаться анонимным.

Схема слепой подписи Чаума. Опишем по шагам схему слепой подписи Чаума.

Действия стороны B.

1. Выбираются секретные случайные большие простые числа Р и Q, равной длины, которые являются секретом. Данные числа образуют модуль N, используемый для нормирования результатов вычислений: N = P*Q.

2. Выбирается случайный открытый ключ Ко такой, чтобы выполнялись условия:

1< Kо ≤ j(N),

НОД(Kо, j(N)) = 1 (числа Kо и j(N) взаимнопросты),

j(N)=(P-1)(Q-1),

где j(N) - функция Эйлера.

3. Далее, используя расширенный алгоритм Евклида, вычис­ляется секретный ключ Kс как обратный элемент к  Kо, такой, что

Kc * Ко ≡ 1 mod j(N) или Kc * Ко-1 mod j(N) = 1.

4. Передача стороне A чисел N, Ko.

Действия стороны A.

5. Документ М, разбитый на n блоков Mi, ,  сжимается с помощью хэш-функции hп(*) в целое число m: m = h(М). В качестве хэш-функции, в рамках данной лабораторной, можно использовать операцию: m = (m Å (MiKo) mod N) mod N, где 0 ≤ m ≤ N-1.

6. Выбирается секретное случайное целое число k, удовлетворяющее условиям: 0 < k < N и НОД(N, k) = 1.

7. С помощью расширенного алгоритма Евклида вычис­ляется k-1 как обратный элемент к k: (k * k-1) mod N = 1.

8. Вычисляется замаскированное сообщение m’ = m kKo mod N и отсылается стороне В.

Действия стороны B.

9.  Вычисляется подпись s’ = (m’)Kc mod N и отправляется стороне А.

Действия стороны A.

10. Вычисляется подпись s = k-1 s’ mod N, которая является подписью В на сообщении m.

Получение слепой подписи базируется на следующих формульных расчетах. Так как (k * k-1) mod N = 1  и (m’)Kc mod N= (m kKo)Kc mod N= k mKc mod N, то k-1 s’ = k-1 (m’)Kc mod N = k-1 k mKc mod N = mKc mod N.

 

Пример. Получить слепую подпись по схеме Чаума.

У пользователя A есть текст «ЭБФ», который он хочет подписать цифровой подписью стороны B.

Действия стороны B.

Пусть заданы простые числа P = 7 и Q = 11. Задан открытый ключ Ko = 13.

Общий модуль N = P*Q = 7*11 = 77. Функция Эйлера от N равна j(N) = (P-1)(Q-1) = 6 * 10 = 60.

Проверка открытого ключа Ko = 13 на соответствие условиям:

1 < Ko ≤ 60  и НОД(Ko, 60) = 1 (алгоритм Евклида).

Вычисляется секретный ключ Kc - обратный элемент для Ko:

Kc = Ko -1 mod j(N) = 13-1 mod 60 = 37.

Передача пользователю A двух чисел N = 77 и Ko = 13.

Действия стороны A.

Получение стороной A двух чисел N = 77 и Ko = 13.

Вычисление хеш. Пусть m = 0. Тогда

- при M0 = 30: m = m ÅM0Ko mod N = (0 Å 3013 mod 77) mod 77 = (0 Å 72) mod 77 = 72.

- при M1 = 2: m = m ÅM1Ko mod N = (72 Å 213 mod 77) mod 77 = (72 Å 30) mod 77= 9.

- при M2 = 21: m = m ÅM2Ko mod N = (9 Å 2113 mod 77) mod 77 = (9 Å 21) mod 77 = 28.

Хэш равно m = 28.

Проверка числа k = 15 на соответствие условиям:

0 < k < 77 и НОД(k, 77) = 1.

Вычисляется обратный элемент для числа k (расширенный алгоритм Евклида):  k-1 mod N =  36.

Находим m’ = (m*(kKo mod N) mod N = (28 * (1513 mod 77)) mod 77 = 28*64 mod 77 = 1792 mod 77 = 21.

Действия стороны B.

Вычисление числа s’ = (m’)Kc mod N = 2137 mod 77 = 21.

Действия стороны A.

Вычисление числа s = (k-1 * s’) mod N = 36*21 mod 77 = 756 mod 77 = 63.

 

Дополнительные примеры: выполните постановку подписи для первых букв своих фамилии, имени и отчества.
