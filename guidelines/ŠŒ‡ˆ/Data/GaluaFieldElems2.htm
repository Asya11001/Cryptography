Алгоритм сложения полиномов

 

На вход данного алгоритма подаются два полинома, представленные в бинарном виде (в стандартном базисе).

Шаги этого алгоритма можно представить в следующем изложении.

1. Вектор, у которого степень полинома максимальна, становится первым операндом, а полином с наименьшей степенью – вторым.

2. Создается промежуточный вектор Temp, с таким же числом строк и столбцов, как у первого операнда. В Temp копируется содержимое первого операнда.

3. Определяется differ - разница в степенях первого и второго операндов.

4. Для , где k2 – длина второго операнда, осуществляется сложение по модулю два элемента (differ+i) первого операнда с элементом i второго операнда и запись результата в элемент (differ+i) вектора Temp.

5. С помощью алгоритма коррекции длины полинома устанавливается новая длина вектора Temp.

6. Далее содержимое вектора Temp, вместе с его параметрами, копируется в первый из полиномов, поступающих на вход данного алгоритма.

Классический алгоритм умножения полиномов с взятием модуля

 

Для работы алгоритма на вход подаются два полинома, представленные в бинарном виде (в стандартном базисе), а также вектор-модуль Modul.

Классический алгоритм умножения полиномов включает в себя следующие действия.

1. Определяется, какой из векторов наибольший и наименьший. Наибольший операнд становится первым операндом, а наименьший – вторым (наименьший операнд имеет меньшее число единичных элементов в векторном представлении).

2. Производится последовательный поиск во втором операнде единиц: , где k2 – длина второго операнда.

а) при нахождении единицы осуществляется суммирование результирующего полинома Rez с первым операндом с того места, где был найден единичный коэффициент;

б) если длина результата Rez больше или равна максимальной степени Modul, то осуществить поразрядное сложение по модулю 2 векторов: Rez= Rez+Modul и обнулить все коэффициенты полинома Rez при степенях, больших степени Modul.

3. Длина результирующего вектора без взятия модуля имеет размерность k1+k2-1, где ki, , - длины первого и второго операндов, а при взятии модуля равна максимальной степени модуля n.

Сложность алгоритма равна Q(3k+4n+kn), где n и k – степени наибольшего и наименьшего полиномов.

Из-за представления данных в виде бинарных векторов усложняется выборка коэффициентов из определенных позиций операндов, так как в ЭВМ минимальной единицей работы с памятью является машинное слово (байт, слово (2 байта), или двойное слово (4 байта)). Также приходится производить сложение побитно, а не целыми машинными словами. Производить операцию сложения векторов, оперируя машинными словами, невозможно, в рамках данного алгоритма, так как позиции, с которых будут производиться сложение, в результирующем векторе могут быть не кратны длине машинного слова.

Данная задача может быть решена в модификациях классического алгоритма умножения полиномов.

Алгоритм арифметического увеличения полинома на единицу

 

Данная функция может применяться в алгоритме проверки полинома на неприводимость (на не делимость полинома на все меньшие по длине другие вектора). Процедура производит арифметическое прибавление единицы на конце вектора с рекурсивным переносом возможного флага переноса на старшие разряды.

Этапы данного алгоритма следующие.

1. Переменной Ostatok присваивается единица.

2. Для  в диапазоне от k до 1, где k – максимальная степень многочлена, производятся действия:

а) складываются значения переменной Ostatok и элемента входного вектора с индексом i, результат сохраняется в переменной Summa;

б) значение переменной Summa берется по модулю два и результат записывается в элемент входного вектора с индексом i;

в) значение переменной Summa делится на два и результат устанавливается в переменную Ostatok;

г) уменьшается на единицу переменная i до тех пор, пока значение Ostatok не станет равным 0. Если Ostatok не равен 0, то осуществляется переход к пункту 2. а).

3. Если значение переменной Ostatok равно единице, то выполняются следующие пункты:

а) увеличивается размер буфера, отводимого для хранения вектора, на единицу (алгоритм увеличения размера данных для полинома от одной переменной);

б) увеличивается на единицу переменная, хранящая максимальную степень полинома;

в) для , где k – новая длина полинома, производится перенос значения векторного элемента с индексом i на место элемента с индексом i+1;

г) на место первого элемента вектора записывается единица.

Данный алгоритм непосредственно изменяет входной вектор. Сложность алгоритма составляет Q(2k), где k – максимальная степень многочлена.

 

Получение примитивных элементов через вычисление

сопровождающей матрицы A

 

Данный алгоритм может использоваться для вычисления множества примитивных элементов.

На вход алгоритма подаются:

- модуль-вектор, задающий размерность поля - Modul;

- степень примитивного элемента, на основе которой необходимо получить значение примитивного элемента - Degree.

Возвращается вычисленное значение примитивного элемента в полиноме PrimEl.

Алгоритм подразделяется на следующие шаги.

1. Осуществляется проверка модуль-вектора на граничные значения: если для данного модуля нельзя построить поле в нормальном базисе, то выход с выдачей ошибки.

2. Если размеры вектора PrimEl малы, то расширить полином до размеров примитивного элемента. Обнуление буфера под данные вектора PrimEl.

3. Если степень примитивного элемента равна неопределенности, то выход с выдачей положительного результата. Иначе пункт 4.

4. Если степень примитивного элемента равна нулю, то в (n-1) позицию (самую младшую часть полинома) вектора PrimEl устанавливается единица; n – размерность полинома-модуля, а нумерация в векторе начинается в нуля. Выход с выдачей положительного результата. Иначе пункт 5.

5. Если степень примитивного элемента равна единице, то в (n-2) позицию  вектора PrimEl устанавливается единица. Выход с выдачей положительного результата. Иначе пункт 6.

6. Получение двух сопровождающих матриц из полинома-модуля на основе алгоритма построения сопровождающей матрицы.

7. Для  осуществляется перемножение сопровождающих матриц с записью результата в первую из них. Перемножение матриц производится с помощью операции умножения алгебры матриц.

8. Из первой сопровождающей матрицы копируется в вектор PrimEl строка (n-1), которая и содержит искомое значение примитивного элемента. Далее осуществляется выход из алгоритма с выдачей положительного результата.

Сложность алгоритма равна Q(n3(2n-1)+n2+3n).

 

Получение множества примитивных элементов на регистрах

 

Данный алгоритм может использоваться для вычисления множества примитивных элементов поля.

На вход алгоритма подается модуль-вектор, задающий размерность поля – Modul.

Возвращается вычисленное множество значений примитивных элементов в матрице полиномов E, где ее i-ый номер строки задает порядковый номер элемента поля, , n – максимальная степень модуля, а в i-ой строке располагается в векторном виде i-ый примитивный элемент. Значения элементов в векторном виде записываются в прямом порядке: слева направо записываются коэффициенты от старших степеней неизвестных полиномиального представления к младшим.

Опишем данный алгоритм по шагам.

1. В нулевую строку матрицы E записывается нулевой вектор, а в первую строку вектор следующего вида 000…001.

2. Для  осуществляются следующие действия:

а) сдвиг (i-1) элемента матрицы E на один разряд влево и запись значения в P;

б) если длина вектора P является большей или равной n, то осуществить поэлементное сложение по модулю 2 вектора P и Modul: P= P+Modul.

в) запись значения вектора P в i-ую строку матрицы E.

Сложность алгоритма равна Q(2n+(3n+1)(2n-2)).
